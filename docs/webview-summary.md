# Открытие ссылок в WebView — итоги проверки

## Что проверяли

В рамках задачи проверили варианты открытия ссылок из WebView:

- Обычные ссылки `<a href="..." target="_blank">` — переход внутри WebView или в браузере в зависимости от нативной обработки.
- Прямые ссылки на PDF — открытие/скачивание PDF по URL.
- Кастомные схемы (`yandexbrowser://`, `googlechrome://`, `browsers://`) — дают **ERR_UNKNOWN_URL_SCHEME**, если натив не перехватывает и не открывает через Intent/open.
- iframe с внешним контентом (zvuk.com и др.) — при отключённом JS внутри iframe скролл и кнопки не работают.
- postMessage, скрипты — не работают при выключенном JavaScript в WebView.
- Локальный PDF в проекте (`public/30day_dec_prime.pdf`) — корректно открывается и скачивается по ссылке с `download` (same-origin).

## Основная проблема

На стороне мобильной разработки:

1. **Отключён JavaScript** в WebView — скрипты, postMessage и интерактив внутри iframe не работают.
2. **Нет обработки ошибок и перехвата навигации** — кастомные схемы приводят к ERR_UNKNOWN_URL_SCHEME, ссылки с `target="_blank"` не обязательно открываются в браузере.

Из-за этого не работают: диплинки браузеров, открытие PDF в браузере по клику, любой сценарий, завязанный на JS или на нативный перехват URL.

## Решение, которое может быть

**Не вести пользователя на прямой URL PDF**, а использовать **ссылку на страницу Zvuk с документом (соглашением/условиями)**. Остальные варианты в текущих ограничениях (без JS, без доработки натива) ненадёжны или не работают.

Пример:

```html
<a href="https://zvuk.com/about/documents/terms" target="_blank" rel="noopener">PDFtest</a>
```

Пользователь переходит на ресурс Zvuk (например, страница с документом или соглашением), где уже можно просмотреть или скачать PDF в рамках сайта. Так ссылка открывается в WebView или в браузере без зависимости от прямого PDF-URL и без необходимости JS/нативной обработки ошибок.

**Итог:** в текущих условиях (WebView без JS, без перехвата нативной навигации) использовать ссылку на страницу Zvuk с документом, а не прямую ссылку на PDF.
